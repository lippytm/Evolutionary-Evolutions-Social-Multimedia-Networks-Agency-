# ✅ Monetisation Checklist — Evolutionary Evolutions Agency

## Overview

Use this checklist to systematically activate revenue streams across every repository and platform. Work through it once for the agency as a whole, then repeat the per-repository section for each new repo you create.

---

## 🏢 One-Time Agency Setup (Do Once)

### GitHub Profile & Sponsors
- [ ] Complete GitHub profile: bio, avatar, website link, location
- [ ] Apply for [GitHub Sponsors](https://github.com/sponsors)
- [ ] Set up at least 3 sponsor tiers ($5, $25, $100)
- [ ] Write a compelling sponsor page description
- [ ] Add FUNDING.yml to the organisation (`.github/FUNDING.yml`)
- [ ] Pin your Sponsors link in your GitHub bio

### Social Media Presence
- [ ] Create/claim consistent username on: Twitter/X, LinkedIn, YouTube, TikTok, Instagram
- [ ] Write professional bios that mention your technical focus (blockchain, robotics, AI)
- [ ] Add GitHub profile link to all social bios
- [ ] Create a simple link-in-bio page (e.g., [Linktree](https://linktr.ee))

### Email List / Newsletter
- [ ] Sign up for a newsletter platform: [Beehiiv](https://beehiiv.com), [Mailchimp](https://mailchimp.com), or [ConvertKit](https://convertkit.com)
- [ ] Create a lead magnet (e.g., "Free Blockchain Starter Kit" or "Top 10 Robotics Libraries")
- [ ] Add subscribe link to all README files and social bios
- [ ] Send first welcome email to subscribers

### Digital Products
- [ ] Create a [Gumroad](https://gumroad.com) or [Lemon Squeezy](https://lemonsqueezy.com) account
- [ ] Set up [Stripe](https://stripe.com) for direct payments
- [ ] Plan your first product (course, eBook, template bundle)
- [ ] Build a landing page for the product

### Consulting
- [ ] Create a "Hire Me" or "Work With Us" page (can be a GitHub page)
- [ ] List your services and pricing
- [ ] Add a booking link (e.g., [Calendly](https://calendly.com))
- [ ] Create profiles on [Upwork](https://upwork.com) or [Toptal](https://toptal.com)

## 📁 Per-Repository Checklist (Repeat for Each Repo)

### Discoverability
- [ ] Write a clear, keyword-rich repository description (1–2 sentences)
- [ ] Add 5–10 relevant GitHub Topics (tags) to the repo
- [ ] Create a comprehensive README.md (see template in [CONTENT_STRATEGY.md](CONTENT_STRATEGY.md))
- [ ] Add a `LICENSE` file (MIT is developer-friendly)
- [ ] Add a `CONTRIBUTING.md` to invite contributors

### Monetisation Hooks
- [ ] Add a "Sponsor this project" button (via FUNDING.yml)
- [ ] Add "Support / Sponsor" section to the README
- [ ] Include affiliate links where relevant tools are mentioned (disclosed)
- [ ] Add a link to your course or digital product if related

### Content Pipeline
- [ ] Create at least one tutorial based on this repository (dev.to or blog)
- [ ] Record a demo video and upload to YouTube
- [ ] Share on Twitter/X with relevant hashtags
- [ ] Submit project to a relevant "awesome-*" curated list
- [ ] Open at least 2 "good first issue" tickets for contributors

### Community
- [ ] Enable GitHub Discussions
- [ ] Respond to all Issues within 48 hours
- [ ] Thank all contributors in Release notes

## 📺 YouTube Channel Monetisation Checklist

- [ ] Create YouTube channel with consistent branding
- [ ] Upload 10+ videos before promoting heavily
- [ ] Reach 1,000 subscribers and 4,000 watch hours → Apply for monetisation
- [ ] Set up [channel memberships](https://support.google.com/youtube/answer/7544492) (exclusive content for members)
- [ ] Add [Super Thanks and Super Chat](https://support.google.com/youtube/answer/10332365) for live streams
- [ ] Reach out to 3 potential sponsors once channel has consistent views
- [ ] Add Patreon or GitHub Sponsors link to all video descriptions

## 🔗 Affiliate Programme Checklist

- [ ] Sign up for at least 3 affiliate programmes (see [REVENUE_GUIDE.md](REVENUE_GUIDE.md))
- [ ] Create a `TOOLS.md` or "Resources" section listing recommended tools with affiliate links
- [ ] Add FTC disclosure statement where affiliate links are used
- [ ] Track clicks monthly using UTM parameters or affiliate dashboard

## 📊 Monthly Revenue Review (Do Monthly)

- [ ] Update the revenue tracking table in [REVENUE_GUIDE.md](REVENUE_GUIDE.md)
- [ ] Review which content drove the most GitHub stars/traffic
- [ ] Follow up with any open sponsorship conversations
- [ ] Identify the top-performing affiliate link and create more content around it
- [ ] Plan next month's content using the [Content Calendar](CONTENT_CALENDAR.md)
